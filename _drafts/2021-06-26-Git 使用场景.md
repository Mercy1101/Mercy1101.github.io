# Git使用场景

* 我们在提交commit的时候， 我在想什么
* 我们在哪里卡壳了
* 首先介绍 commit

我们使用commit记录代码的修改，使用 revert commit 撤销commit
来完成什么样的场景

## 场景1： 我们要对代码进行记录版本
我们可以使用commit来记录记录代码的版本和信息,如果想要回退可以使用，git reset --hard

## 场景二： 我们要开发一个功能，
从develop分支上拉下来最新的代码,开发过程中，想在不影响develop上的功能的同时，把代码同步到另一台电脑上。
可以自己从develop分支上建立个分支，开发的过程代码，提交到自己的分支上，该分支上传至远端，不影响其他人使用develop

##  场景三： 开发一个功能开发了一半，帮我在版本上传一个东西，我现在就要

我想要切换回master分支，出一个版本，但是没有时间给你commit

开发过程中，想切换到其他分支做修改，但发现切换分支需要，工作区是干净的，即把所有未commit的代码都commit上。我们可以把修改过的代码，放到一个临时的地方，让我们的工作区变得可以切换分支，等完成工作后切换回来自己的分支后，我们可以把再把这些修改过的代码拿出来。
```test
git stash   ///< 草稿导出

/// work space clean

git checkout develop

/// 做一些事情, 再切换回来
git checkout future
git stash pop   ///< 把刚才存放的草稿，导入到当前分支
```

## 我之前的提交信息写错了，怎么改
修改最近一次提交的commit
```git
 git commit --amend
```
如果想要修改之前的commit
```git
git rebase -i HEAD~3    ///< 回退到HEAD前面第三个commit处
# Rebase 578c0a3..4d6a29a onto 578c0a3 (3 commands)
#
# Commands:
# p, pick <commit> = use commit
# r, reword <commit> = use commit, but edit the commit message
# e, edit <commit> = use commit, but stop for amending
# s, squash <commit> = use commit, but meld into previous commit
# f, fixup <commit> = like "squash", but discard this commit's log message
# x, exec <command> = run command (the rest of the line) using shell
# b, break = stop here (continue rebase later with 'git rebase --continue')
# d, drop <commit> = remove commit
...

想要修改哪一个提交就把pick换成你想要的操作,如edit
然后执行
git rebase --continue
```

## 打tag
```git
git tag -a vx.x.x -m "message"
git push origin tags    ///< 推送到远端
```

## 合并分支
```git
git checkout future
git merge develop   ///< 把develop的东西合并到当前分支
git checkout develop
git merge future    ///< 把future合并到当前分支
git branch -d future    ///< 把合并过的分支删除
```
